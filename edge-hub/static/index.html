Settings

Connect this hub to your GDT Console and the RS485 adapter. Use Save settings at the bottom of this page to apply changes to connection, serial, and timing.

Typical setup

  1. Here: paste Console URL + API key, choose serial port, save.
  2. Site: load sites and pick one (loads reactors).
  3. Reactors: open each reactor → add sensors; each add saves immediately.
  4. Calibration: optional fast Modbus polling for pH calibration (uploads follow Settings).
  5. Live: start monitoring.

Connection

Serial / Modbus

USB‑RS485 adapters often appear as COM3, COM4, … on Windows or /dev/ttyUSB0 on Linux.

Polling & upload

Intervals are in minutes (decimals allowed). Example: poll 0.02 ≈ 1.2 s; upload 1 = once per minute.

Apply settings

Saves URL, key, serial port, baud, and intervals. Does not affect reactor devices (those save when you add or remove them).

Local reactors & sensors

Calibration mode

For pH calibration, use faster sampling so you can compare readings against buffer standards. One button switches Modbus polling to every 5 seconds while uploads keep using your Settings interval, then restores your saved poll interval when you turn it off.

Live data & upload queue

Start monitoring to poll Modbus and stream data. If the network drops, readings queue locally (see the bar at the top) and upload when the connection returns.
